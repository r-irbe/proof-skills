## Methodology

- **Solver protocol** — short-prompt convention (≤8 sentences, no
  preamble) reused from R18 Phase 18; 36 task prompts replayed verbatim.
- **gpt-5.4 dispatch quirk** — gpt-5.4 sub-agents cannot read or write
  `/tmp/...` paths ("environment security restrictions"). Mitigation:
  inline task payload via `write_agent` on turn 2; `read_agent
  since_turn=1` to skip the turn-0 refusal.
- **Judge** — single opus-4.7-high judge per case (R18 single-judge
  convention), strict-JSON `{score, rationale, rubric_evidence}` schema.
  Same rubric routing as R18 (`lean-proof-quality`, `lean-doc-quality`,
  `lean-setup-import-quality`, `applied-domain-quality`).
- **Match conversion** — 36 cases × 3 pairwise opponents (sonnet, opus,
  haiku) = 108 new rows. Score comparison: higher = win, equal = draw.
- **Reproducer** — `/tmp/round19/g54_judge_build.py` (prompt builder),
  `/tmp/round19/g54_elo_build.py` (CSV writer).

## Observations

1. **gpt-5.4 ranks 3rd of 4**, consistent with prior public benchmarks
   placing it below the Claude flagships but above haiku-tier models.
2. **12 cases tie all four entrants at score 1** (all the lean-*
   short-prompt cases asking for Lean code where every model produced
   prose-only outputs). These are case-design problems, not solver
   differentiation — they add no signal to ELO updates (all draws).
   Skill-pruning candidate set unchanged from R18.
3. **gpt-5.4 ties or beats opus** on 21 of 36 cases (58%); ties or
   beats sonnet on 17/36 (47%); ties or beats haiku on 30/36 (83%).
   No case where gpt-5.4 outscores both Claude flagships at once.
4. **One notable miss** — `lean-mwe-smoke`: gpt-5.4 produced a Lean
   #guard_msgs example that intentionally fails (`exact "x"` against
   `Nat`), correctly demonstrating the MWE pattern but not discharging
   a goal. Judge scored 2 vs opus's 5. Rubric punishes literal "doesn't
   discharge goal" even when the task asks for a failing MWE — case
   design quirk worth flagging.
5. **Regression gate (75-pt tolerance) passes**: Δopus +20, Δsonnet
   −3, Δhaiku −7. No baseline refresh needed.

## Operational implications

- The 4-way leaderboard is stable enough to publish: 108 games for
  gpt-5.4 gives ±36 CI, well within Phase 1 design tolerance.
- gpt-5.4 sandbox quirk (no `/tmp` access) is durable; future gpt-5.4
  dispatches must use the inline-payload pattern documented in
  `scripts/eval/graders/DISPATCH.md` (see Flavour A note on file
  paths).
- Skill-pruning audit on the 12 all-1 lean-* cases is still pending
  (HITL gate from R19); the gpt-5.4 evidence reinforces that these
  cases test task-comprehension calibration rather than solver
  capability.
