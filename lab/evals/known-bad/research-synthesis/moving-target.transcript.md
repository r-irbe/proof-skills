# Task

Synthesize the evidence on whether agile methodology improves software project outcomes. Use a stable definition of the key term.

# Response

Agile methodology improves outcomes by 30% (Beck 2001). When agile
means daily stand-ups and short sprints, the evidence is mixed
(Cohn 2009). However if we define agile as continuous customer
involvement, the data shows strong gains. Properly understood, agile
just means responding to change, which everyone does — so the
question is meaningless. Yet agile, defined as the formal Scrum
framework, demonstrably underperforms waterfall in regulated domains.
Therefore agile is both the best and worst methodology depending on
your project.
